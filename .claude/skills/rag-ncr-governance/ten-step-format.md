# 규정/NCR 답변 10단계 정본 형식

> 정본 = `CLAUDE.md §10`. `docs/08`·`docs/18`은 이에 맞춰 정렬한다. 규정·NCR 관련 **모든** 답변은 아래 10단계를 따르고, 항상 **"검토용 초안"**(공식 법규 해석 아님)을 명시한다.

## 10단계 (정본, `CLAUDE.md §10`)

1. **질문 요약** — 무엇을 묻는지 1~2줄.
2. **적용 기준일** — 판단 기준이 되는 날짜(검색 기준일과 일치). 주입 `IClock` 실제 날짜, placeholder 금지.
3. **적용 문서/규정 버전** — 문서명 + 버전 + 시행일(공개 catalog 메타 기준).
4. **관련 조항 또는 내부기준** — 조항/문단 번호(원문 전재 아님, 참조).
5. **업무 적용 판단** — 리스크관리 업무 관점의 적용 여부·범위.
6. **필요 데이터** — 대사·검증에 필요한 더미 컬럼만(`RISK_EXPOSURE_DAILY`/`RISK_LIMIT_MASTER` 등 일반 더미).
7. **SQL/VBA 또는 Excel 검증 방법** — 조회 전용 SQL/검증 절차. 자동 실행 0.
8. **리스크관리 관점 주의사항** — Hidden Risk(구버전 산식·기준일 불일치·매핑 오류 등).
9. **준법/리스크관리 확인 필요사항** — 문서오너/준법 확인이 필요한 항목.
10. **출처** — 문서명·버전·시행일·조항·출처(locator)·검색 기준일.

> 모든 답변 말미(또는 머리)에 **"본 답변은 검토용 초안이며 공식 법규 해석이 아님"** 고정 문구.

## 인용 블록 필수 항목 (`docs/17` 검색 답변 표시)

검색·인용 답변에는 다음 7항목이 **모두** 있어야 한다(하나라도 누락 시 위반):

```text
문서명 · 버전 · 시행일 · 조항 · 출처 · 검색 기준일 · "검토 필요" 문구
```

- `버전`/`시행일`/`file_hash`가 repo에 비어있으면(원문 미포함) `(확인 필요)`로 노출 — placeholder 값을 채우지 않는다.
- `출처`는 출처기관이 아니라 **locator**(문서 내 위치 참조).

## NCR 답변 추가 구조 (`docs/08`·`docs/18`)

NCR은 위 10단계 위에, Rule Set 8요소 구조에서만 설명한다(**모델 산식 암기 0**):

```text
NCR Rule Set · Rule Set Version · Effective Date · Component Map
Formula Description(서술적) · Validation SQL Template(조회 전용)
Regulation Basis(출처) · Approval History
```

- 산출은 Validation SQL/대사로 **재현·감사 가능**해야 한다.
- NCR 공식본 원문·실 계수는 repo 미포함 — Prod 권한통제 KB로만(`CLAUDE.md §10`).

## 절대 금지 (이 형식 적용 시)

- 내부규정 원문 · NCR 공식본 원문 전재 = **금지**.
- 실 테이블/컬럼/시스템명·실 데이터·secret·모델파일 = **금지**(일반 더미만).
- 검색 기준일·시행일에 **임의 placeholder 날짜 하드코딩 금지**.
- "공식 해석"으로 단정 금지 — 항상 검토용 초안.
