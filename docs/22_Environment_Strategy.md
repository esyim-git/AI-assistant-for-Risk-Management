# Environment Strategy

## 최종 환경 정의

```text
GitHub / 개발 PC      = 개발환경 Dev
Local 실행 PC         = 테스트환경 Test
회사 업무망 PC         = 운영환경 Prod
회사 개발망 PC 포함    = 운영환경처럼 취급
```

## 핵심 결정

회사 PC는 개발망이라도 운영환경처럼 취급한다.
즉, 회사 PC에서 개발도구 설치, 인터넷 패키지 복원, 외부 API 호출을 기본 전제로 하지 않는다.

## Dev

역할:

- 소스코드 관리
- Claude Code/Codex 활용
- 빌드
- 테스트
- Release ZIP 생성

포함 가능:

- 더미 데이터
- 공개자료 catalog
- 구조/문서/템플릿

포함 금지:

- 회사 실데이터
- 내부규정 원문
- 실제 테이블 사전
- 계정정보

## Test

역할:

- Release ZIP 실행 검증
- 업무망 반입 전 오프라인 동작 확인
- 더미 데이터 테스트

테스트 조건:

- 인터넷 없이 실행
- 개발도구 없이 실행
- 샘플 데이터 분석 가능
- 모델 없어도 앱 기동

## Prod

역할:

- 승인된 Release ZIP 실행
- 실제 업무 보조
- 내부규정/NCR/테이블 사전은 운영환경 안에서 별도 승인 후 구성

금지:

- 소스 빌드
- 외부 다운로드
- 자동 업데이트
- 외부 API 호출
- GitHub 직접 접속 의존

## 배포 흐름

```text
Dev에서 Release ZIP 생성
  ↓
Test에서 ZIP 검증
  ↓
반입 신청/해시 확인
  ↓
Prod에 압축 해제
  ↓
운영 실행
```
