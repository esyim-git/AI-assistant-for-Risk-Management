# Security Policy

## 기본 방침

이 프로젝트는 망분리 금융회사 환경을 전제로 한다.

## 금지

- 운영 DB 자동 실행
- VBA 자동 실행
- 외부 API 호출
- 자동 업데이트
- Telemetry
- 비밀번호 저장
- 회사 데이터 Repository 포함
- 내부규정 원문 Repository 포함
- 모델 파일 Repository 포함

## 허용

- 더미 데이터 테스트
- 공개자료 카탈로그
- 룰 기반 검사
- SQL/VBA 초안 생성
- 검증 SQL 생성
- 사용자 수동 실행
- 작업 로그 저장

## 운영환경 로그 원칙

로그에는 다음을 저장하지 않는다.

- 고객명
- 계좌번호
- 비밀번호
- 접속정보
- 원문 내부규정
- 대량 운영 데이터

로그에는 다음을 저장할 수 있다.

- 작업 ID
- 사용자 ID 또는 사번 hash
- 기능명
- 룰 검사 결과
- 템플릿 버전
- 모델 버전
- 결과 hash
